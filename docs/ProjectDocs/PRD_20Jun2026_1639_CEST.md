## 1. Product Goals & Scope

The **USB Speed Utility & Monitor** is a desktop application written in Python (using `pywebview` for a lightweight HTML/JS/CSS frontend) that benchmarks USB drive read/write performance, lists active peripherals, monitors storage capacities, and displays comparative reports. 

The goal of this product release is to integrate an **AI Chatbot Assistant** that functions entirely locally or connects to cloud LLM providers via user-provided API keys.

---

## 2. User Stories

1. **AS A** non-technical user,  
   **I WANT TO** paste my OpenAI / Anthropic key or connect to my local Ollama instance,  
   **SO THAT** I can ask an AI assistant to analyze my USB speed test results without paying for a subscription.

2. **AS A** hardware tester,  
   **I WANT TO** quickly click a button like "Analyze Speeds" in the chat interface,  
   **SO THAT** the AI automatically receives the details of my connected USB drives and latest speed test run and diagnoses any performance bottlenecks.

3. **AS A** privacy-conscious user,  
   **I WANT** my API keys and conversation logs to be stored locally in the installed folder or user configuration directory,  
   **SO THAT** my credentials are never uploaded to any analytics backend.

## 3. Functional Specifications

### 3.1 AI Assistant Tab
*   **Chat Viewport**: A scrollable history containing message blocks. It must distinguish between the User (aligned right, primary theme color) and the Assistant (aligned left, secondary dark background).
*   **Markdown Rendering**: The UI must render assistant replies with support for basic Markdown formatting (bold text, bullet lists, tables, and code blocks with scrollbars).
*   **Quick Prompts Panel**: A set of clickable quick-action buttons:
    - *“Analyze Latest Benchmark”*: Sends current speed test data.
    - *“Explain USB Standards”*: Provides a reference guide on USB speeds.
    - *“Troubleshoot Slow Speeds”*: Starts a troubleshooting guide for slow writes.
*   **Interactive Input**: A text area with `Enter` (or `Ctrl+Enter`) key-bindings to send, and a button to send the message.
*   **Clear Chat**: A button to wipe the current chat history from the screen.

### 3.2 LLM Settings and Configuration
The **Settings Modal** will contain an **AI Chatbot Configuration** section with the following inputs:
*   **AI Provider Selection**: A dropdown containing:
    1. `Ollama (Local)` - Default, requires no API key.
    2. `Anthropic Claude` - Requires Anthropic API key.
    3. `OpenAI` - Requires OpenAI API key.
    4. `Custom (OpenAI-compatible)` - For custom API URLs (e.g. LM Studio, LocalAI).
*   **API Key Input**: Password-masked text input field for storing cloud provider keys. Includes a "Show/Hide Eye" icon.
*   **Model ID Input**: Text input to specify the target model (e.g. `llama3` for Ollama, `gpt-4o` for OpenAI, `claude-3-5-sonnet-20240620` for Anthropic).
*   **Endpoint URL Input**: Specifies the base URL. Pre-filled based on provider choice:
    - Ollama: `http://localhost:11434`
    - OpenAI: `https://api.openai.com`
    - Anthropic: `https://api.anthropic.com`
    - Custom: Editable empty value.
*   **Test Connection Button**: Triggers a backend connection check to verify the LLM replies to a simple greeting.

### 3.3 Context Injection Mechanism
When a message is sent to the backend API `send_chatbot_message(chat_history)`:
1. The Python script retrieves the active system state:
   - Connected USB devices list (models, protocols, capacities).
   - The last benchmark test results (read speed, write speed, test file size, duration).
2. The Python script injects a **System Prompt** at the beginning of the context:
   ```text
   You are the USB Speed Utility AI Assistant. You help users analyze USB drive performance, explain USB technical specs, and troubleshoot issues.
   Current system details:
   - Connected USB Devices: [List of devices]
   - Last Benchmark: [Details of last run speed test]
   Provide helpful, technical, yet user-friendly advice.
   ```
3. The conversation payload is sent to the target LLM provider API.

### 3.4 Local Storage Requirements
*   All configuration data is saved in `config.json`.
*   **Portable Mode Rule**: If the application's root directory is writable (portable installation/source mode), `config.json` is stored inside the application directory. If the directory is write-protected (e.g., `C:\Program Files`), the configuration defaults to `C:\ProgramData\UBSSpeedTest` (Windows) or user's standard home config directories (macOS/Linux).
*   API keys must be stored under the `"ai_chatbot"."api_key"` key.

## 4. Non-Functional Requirements

### 4.1 Dependency Limits
*   **Zero New Native Packages**: No external pip packages (like `openai` or `anthropic`) will be added to the project. The connections will be made using Python's built-in `urllib.request` library. This guarantees PyInstaller compiles a lightweight, self-contained binary.

### 4.2 Security & Data Privacy
*   **Direct API Connections**: The desktop app must speak directly to Ollama or the selected LLM provider endpoint. There must be no intermediate proxy servers.
*   **Credential Masking**: API Keys must never be displayed in plain text in the settings UI after being saved, unless requested via a "Show" action.

### 4.3 Error Handling
*   If the selected model is not loaded in Ollama, return a clean error: `"Ollama model [model_name] not found. Please run 'ollama pull [model_name]' in your terminal."`
*   If the API key is incorrect or empty, return: `"Authorization error. Please check your API key in settings."`
*   If offline, return: `"Connection failed. Please check your network connection or local server."`
