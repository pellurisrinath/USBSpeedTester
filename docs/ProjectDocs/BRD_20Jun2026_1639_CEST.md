## 1. Executive Summary

The **USB Speed Utility & Monitor** is a lightweight, cross-platform desktop application designed to provide users with visual diagnostics of their connected USB peripherals, write/read speed benchmarks for USB storage drives, real-time disk space monitoring, and comparative performance reports. 

This document defines the business goals, market drivers, and high-level requirements for expanding the product's capabilities. Specifically, the introduction of an **integrated, local-first AI Assistant** that allows users to connect their own LLMs (such as Ollama, OpenAI, or Anthropic Claude) using their own API keys. This feature adds immense user-assistance value without increasing ongoing infrastructure costs for the software publisher.

---

## 2. Business Objectives & Project Value

### 2.1 Problem Statement
1. **Interpretation Barriers**: Technical speed benchmarks (e.g., "45 MB/s write speed") are hard for non-technical users to evaluate. They frequently do not know if their hardware is underperforming or meeting its rated standard (USB 2.0 vs 3.0).
2. **Support & Troubleshooting Overhead**: Users encounter issues like "drive not recognized," "read-only disk errors," or "unsupported file systems" and look for external support, causing user frustration.
3. **Data Loss Risks**: Low storage space on active USB storage drives can cause corrupted files or failed operations. Real-time background monitoring mitigates this, but users need help optimizing their storage.

### 2.2 Business Goals
*   **Zero-Infrastructure AI Support**: Offer robust AI diagnostic features to users without maintaining expensive backend LLM servers or subscription models. This is achieved by enabling **Bring-Your-Own-Key (BYOK)** and supporting local offline LLMs (via Ollama).
*   **Enhanced User Engagement**: Elevate the tool from a simple benchmark utility to an interactive hardware diagnostics platform.
*   **Reduced Support Load**: Empower users to self-solve issues such as formatting drives (FAT32 vs exFAT vs NTFS) and identifying fake/counterfeit USB flash drives by chatting with the integrated AI.

---

## 3. Target Audience & Stakeholders

*   **System Administrators & IT Professionals**: Require quick, reproducible USB speed tests and device audits across corporate workstations.
*   **Casual Desktop Users**: Need an intuitive way to test if a newly purchased USB drive performs as advertised and to get help troubleshooting simple connection issues.
*   **Hardware Enthusiasts**: Compare multiple drives to purchase the best storage medium for specific applications (e.g. Raspberry Pi boot drives, camera cards).
*   **Product Owner / Developers**: Need a maintainable, light codebase that compiles into a single standalone executable.

## 4. Key Business Requirements (BR)

### BR-001: Bring-Your-Own-LLM (BYOLLM) Configuration
The system must allow the user to connect their own LLM backend. 
- Supported backends must include **Ollama (local/offline)**, **OpenAI Cloud**, **Anthropic Claude**, and **Custom OpenAI-compliant APIs** (e.g., LM Studio, local gateways).

### BR-002: Secure Local Storage of Credentials
To respect user privacy and security, all API keys, endpoints, and model configurations must be stored locally on the user's computer inside the application's directory (portable configuration mode) or standard user profile folders. Credentials must **never** be sent to any third-party server other than the configured LLM API endpoint.

### BR-003: Context-Aware Hardware Support
The AI assistant must be contextualized. It should not be a generic chatbot; it must automatically receive information about the user's current USB configuration and latest speed tests (when requested) to provide specific, actionable diagnostics.

### BR-004: Clean Packaging and Portability
The software must compile into a single-file executable that does not require installing Python or external dependencies (like Node, Electron, or large SDK packages). It must run out-of-the-box on standard Windows environments.
